# Contributing to train_kotha

We welcome contributions to the train_kotha project! Please follow the guidelines below to help us maintain the quality of the codebase.

## Fork and Clone
1. Fork the repository by clicking on the "Fork" button at the top-right corner of the page.
2. Clone your forked repository:
   ```
   git clone https://github.com/your_username/train_kotha.git
   cd train_kotha
   ```

## Creating a Feature Branch
Please create a new branch for your feature:
```bash
git checkout -b feature/your-feature-name
```

## Commit Guidelines
- Write descriptive commit messages.
- Use the present tense for commit messages (e.g., "Add new feature" instead of "Added new feature").
- Limit your commit messages to 72 characters.

## Testing Requirements
- Ensure that all existing and new tests pass.
- Run tests using:
```bash
npm test
```

## Code Style Guidelines
- Follow the coding standards outlined in the project's documentation.
- Use consistent naming conventions.
- Ensure code is well-documented.

Thank you for your contributions!
